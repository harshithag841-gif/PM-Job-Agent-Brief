03 · The five big decisions

Every choice is a trade-off.

Five choices carry most of the architecture. Each one had a cheaper alternative that I rejected for a specific reason.

Cloud-first, not local

Runs on GitHub Actions, not on my laptop. My machine can sleep, travel, or be in a hospital recovery room — the agent runs anyway. A local cron would break the moment I closed the lid.

SQLite committed to the repo, not Postgres

The database is tiny, single-writer, and only persists across daily runs. Git gives me free versioning, free backup, zero infra. If it corrupts, I revert one commit. Postgres would be complexity I don't need.

LLM only for the ambiguous tail, not for every job

Rules handle ~70% of decisions deterministically. The LLM earns its quota on the 30% where language ambiguity actually matters. Same logic as a senior reviewer: automate what's obvious, think about what isn't.

Dual-trigger, idempotent

An external cron (cron-job.org) fires first; GitHub's own cron is the backup. Both land in the same workflow, and a pre-flight guard makes double-firing a safe no-op. Belt and suspenders, at zero cost.

Human applies, agent curates

I deliberately didn't build auto-apply. The agent's job is to hand me a clean shortlist; the application itself stays my decision. Faster-but-worse isn't the goal — better-and-faster is.

04 · Scope

What's in, what's deliberately out.

Drawn on day one, held through seven days of building.

In scope

  • Reading across multiple job platforms every day
  • Filtering out stale, irrelevant, out-of-context roles
  • Scoring against the user's real resume and preferences
  • Remembering what was seen before
  • Delivering a clean, ranked shortlist by email

Out of scope

  • Auto-applying on the user's behalf
  • Writing cover letters or editing resumes
  • Impersonating the user in recruiter conversations
  • Replacing the judgement of the person searching
  • Scaling to many users — this is a personal tool, not a platform
05 · Fork this for yourself

Same bones, different person.

If you want to adapt this for a different role or domain, four things need to change. The architecture itself stays.

Swap 1 · Profile

Your resume keywords, years, domain, deal-breakers

The scorer reads this as ground truth. Everything below it derives from this file.

Swap 2 · Platforms

Keep the sources that match your market

Designers: Dribbble, Coroflot. Engineers: HN Who's Hiring, Arc. Add one scraper at a time — failure isolation keeps the rest of the run safe.

Swap 3 · Scoring weights

Tune the auto-HIGH and auto-DROP ladders

Which title patterns matter to you, which geos you'll never relocate to, which seniority tier you belong in. A week of real output tells you where your thresholds are wrong.

Swap 4 · Delivery

Excel-in-email, Notion, Slack, Telegram

The delivery boundary is clean — one interface in, any channel out. Pick the surface you'll actually read at 08:15.
